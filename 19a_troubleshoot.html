Troubleshooting your customized Gibbs sampler
Denis Valle
July 2020

Why is it important to compartmentalize my code?

Because we are creating a lot of code, it is critical to generate some fake data to check if we are getting sensible results. If results do not make sense, we need to figure out exactly what is going on. Ironically, finding the bug(s) is often much more time consuming than writing the algorithm itself!

It is often useful to compartmentalize the Gibbs sampler code by writing functions, typically one for each FCD. Here is the same example as before but compartmentalized into 2 functions:

  • sample.mu()
  • sample.sig2()
#function to sample mu parameter
sample.mu=function(sig2){
  prec=(n/sig2)+(1/sig2_0)
  var1=1/prec
  media=var1*((sum(X)/sig2)+(mu0/sig2_0))
  rnorm(1,mean=media,sd=sqrt(var1))
}

#function to sample sig2 parameter
sample.sig2=function(mu){
  a1=a+(n/2)
  ssq=sum((x-mu)^2)
  b1=b+(ssq/2)
  1/rgamma(1,a1,b1)
}

#core of gibbs sampler
param=list(mu=mu,sig2=sig2)
for (i in 1:ngibbs){
  #keep track of iterations
  print(i)

  #sample mu parameter
  param$mu=sample.mu(param$sig2)

  #sample sig2 parameter
  param$sig2=sample.sig2(param$mu)

  #store results
  store.param[i,]=c(param$mu,param$sig2)
}

As you will soon notice once we start working on multiple examples, the code for the Gibbs sampler can be highly compartmentalized. In other words, we typically have one function for each parameter in our model and we iterate the sampling of each one of these parameters. This might seems just like a cosmetic change but it is not, particularly as the code starts to get complex. This is convenient from the perspective of troubleshooting your code because we can test these functions, one at a time.

The key idea here is to set all the parameters to their true values (these are known since we generated the data) except for the parameter that corresponds to the function that will be tested. If the outcome from this function (given everything else is known and set to their true values) gives correct answers, then we can assume that it is working properly and move on to test the next function.

For instance, we can simulated fake data for which we know the true \(\sigma^2\) and \(\mu\) value and we can test our functions in the following way:

  • test if our function sample.mu is working properly: comment out param$sig2=sample.sig2(param$mu) and include param$sig2=sig2.true.
  • test if our function sample.sig2 is working properly: comment out param$mu=sample.mu(param$sig2) and include param$mu=mu.true.
