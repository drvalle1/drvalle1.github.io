Example of the Metropolis-Hastings (MH) algorithm
Denis Valle
June 2019

Some equations

Going back to our initial example, notice that:

\[log(T(\theta_{new}))=log([\prod_i Binomial(y_i|n_i,\theta_{new})]Beta(\theta_{new}|a,b))\] \[=\sum_i log[Binomial(y_i|n_i,\theta_{new})] + log[Beta(\theta_{new}|a,b))]\]

Also note that, given that we are going to use a truncated normal distribution for our proposal distribution Q(), we have that:

\[Q(\theta_{new}|\theta_{old})=\frac{\frac{1}{\sqrt{2\pi \sigma^2_{jump}}} exp(-\frac{1}{2\sigma^2_{jump}}(\theta_{new}-\theta_{old})^2)}{\Phi(\frac{H-\theta_{old}}{\sigma^2_{jump}})-\Phi(\frac{L-\theta_{old}}{\sigma^2_{jump}})}\] where H and L are the upper and lower truncation points and \(\Phi()\) is the standard normal CDF. As described before, the numerator is going to disappear when we calculate the ratio \(\frac{Q(\theta_{old}|\theta_{new})}{Q(\theta_{new}|\theta_{old})}\). Therefore, we have that:

\[log[\frac{Q(\theta_{old}|\theta_{new})}{Q(\theta_{new}|\theta_{old})}]=log[\frac{\Phi(\frac{H-\theta_{old}}{\sigma^2_{jump}})-\Phi(\frac{L-\theta_{old}}{\sigma^2_{jump}})}{\Phi(\frac{H-\theta_{new}}{\sigma^2_{jump}})-\Phi(\frac{L-\theta_{new}}{\sigma^2_{jump}})}]\] \[=log[\Phi(\frac{H-\theta_{old}}{\sigma^2_{jump}})-\Phi(\frac{L-\theta_{old}}{\sigma^2_{jump}})]-log[{\Phi(\frac{H-\theta_{new}}{\sigma^2_{jump}})-\Phi(\frac{L-\theta_{new}}{\sigma^2_{jump}})}]\]

Some code

The two functions that will be essential here are given below. The first one allows us to sample from a truncated normal (this is useful since this will be our proposal distribution). This function requires the user to specify:

  • lo: lower bound;
  • hi: upper bound;
  • mu: mode of the truncated normal distribution; and
  • sig: \(\sigma^2_{jump}\) (i.e., the width of our proposal distribution).
tnorm <- function(lo,hi,mu,sig){   #generates 1 truncated normal random variable based on cumulative normal distribution
  
  #normal truncated lo and hi
  q1 <- pnorm(lo,mu,sig) #cumulative distribution
  q2 <- pnorm(hi,mu,sig) #cumulative distribution
  
  z <- runif(1,q1,q2)
  z <- qnorm(z,mu,sig)
  if (z==-Inf) z=lo
  if (z==Inf)  z=hi
  z
}

The second function does the accept-reject calculation, after we provide it with:

  • log.pold: \(log(T(\theta_{old}))\)
  • log.pnew: \(log(T(\theta_{new}))\)
  • log.prop.ratio: \(log[\frac{Q(\theta_{old}|\theta_{new})}{Q(\theta_{new}|\theta_{old})}]\)
  • param.old: \(\theta_{old}\)
  • param.new: \(\theta_{new}\)
acceptMH <- function(log.pold,log.pnew,param.old,param.new,log.prop.ratio){   #accept for M, M-H
  
  a    <- exp(log.pnew - log.pold + log.prop.ratio)       #acceptance PR
  z    <- runif(1,0,1)
  if (z < a) param.old = param.new #accept
  param.old
}

Using the functions above, contained in an R file called useful functions.R, we have all the ingredients we need to run the random walk MH algorithm. Here is the code for that:

rm(list=ls(all=T))
set.seed(1)

setwd('U:/UF/courses/bayesian course/2016/chapter12')
source('useful functions.R')

#our data
y=c(3,10,3,2,4,5);
n=c(10,10,5,4,5,6);

#hyper-prior parameters for theta
a=b=1

#this is for the proposal distribution
jump=0.3

#initial value
theta=0.5

nsim=10000
store.theta=matrix(NA,nsim,1)

for (i in 1:nsim){
  # print(i)
  
  #old theta
  theta.old=theta
  
  #propose new theta based on truncated normal
  theta.new=tnorm(lo=0,hi=1,mu=theta.old,sig=jump) 
  
  #calculate target distribution based on old theta
  log.pold=sum(dbinom(y,size=n,prob=theta.old,log=T))+dbeta(theta.old,a,b,log=T)
  
  #calculate target distribution based on new theta
  log.pnew=sum(dbinom(y,size=n,prob=theta.new,log=T))+dbeta(theta.new,a,b,log=T)

  #calculate the ratio of the proposal distributions
  num=pnorm((1-theta.old)/jump)-pnorm((0-theta.old)/jump) #numeratorr
  den=pnorm((1-theta.new)/jump)-pnorm((0-theta.new)/jump) #denominator
  log.prop.ratio=log(num)-log(den)
  
  #do we accept theta.new or do we retain theta.old?
  theta=acceptMH(log.pold=log.pold,log.pnew=log.pnew,
                 param.old=theta.old,param.new=theta.new,
                 log.prop.ratio=log.prop.ratio)
  store.theta[i]=theta
}

par(mfrow=c(3,1))
plot(1:nsim,store.theta,type='l',xlim=c(0,100),ylab='Sampled values',xlab='Iterations')
plot(1:nsim,store.theta,type='l',ylab='Sampled values',xlab='Iterations')
acf(store.theta,main='Autocorrelation')

Examining the output of the first 100 iterations illustrates the behavior of this sampler in that we might be accepting new values very sporadically, so that the value for theta remains unchanged for a large number of iterations (top panel). The second panel shows the trace-plot for all 10,000 iterations. Finally, the bottom panel shows the amount of autocorrelation in our sample. This plot reveals that only after we subset every 5th sample or so do we obtain approximately independent samples. Modelers can try to increase the acceptance rate and/or reduce autocorrelation by changing our tuning parameter \(\sigma_{jump}^2\).

To see if this MH sampler really created a good representation of the FCD is tricky because generally we do not know what the FCD should look like. However, in this particular case, we know that it should be a \(Beta(\theta|\sum_i y_i + a, \sum_i (n_i - y_i)+b)\). We find that our approximation using the random walk MH algorithm (black histogram) is pretty close to the expected distribution (red):

par(mfrow=c(1,1),mar=c(4,4,1,1))
hist(store.theta,probability=T,xlab=expression(theta),main='')

z=seq(from=0,to=1,length.out=10000)
y1=dbeta(z,sum(y)+1,sum(n-y)+1)
lines(z,y1,col='red')
