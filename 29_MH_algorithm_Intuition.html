Intuition for the Metropolis-Hastings (MH) algorithm
Denis Valle
October 2019

Intuition

Some intuition of why this algorithm works comes from the equation:

\[p_{thresh}=min(1,\frac{T(\theta_{new})}{T(\theta_{old})} \frac{Q(\theta_{old}|\theta_{new})}{Q(\theta_{new}|\theta_{old})})\] Let’s ignore for a moment the proposal ratio. If the proposed parameter value \(\theta_{new}\) is in an area of higher probability relative to where \(\theta_{old}\) is, then \(\frac{T(\theta_{new})}{T(\theta_{old})}>1\) and thus \(p_{thresh}=1\). This implies that we always accept \(\theta_{new}\) if this is coming from a region with higher probability (i.e., we always accept these up hill moves).

When the proposed parameter value \(\theta_{new}\) is in an area of lower probability relative to where \(\theta_{old}\) is, then \(\frac{T(\theta_{new})}{T(\theta_{old})}<1\) and thus \(p_{thresh}<1\). This implies that we occasionally accept \(\theta^{new}\) even if the new region has a lower probability (i.e., we sometimes accept down hill moves). This feature ensures that we have some (albeit few) samples in areas of low probability.

Demonstration

Somethings are easier to understand when we see them rather than when we hear an explanation about it. So here is some code that allows you to see what the MH algorithm is doing step-by-step. The functions we will require for this to work are contained in a file called “useful functions.R”, given below:

tnorm <- function(lo,hi,mu,sig){   #generates 1 truncated normal random variable based on cumulative normal distribution
  
  #normal truncated lo and hi
  q1 <- pnorm(lo,mu,sig) #cumulative distribution
  q2 <- pnorm(hi,mu,sig) #cumulative distribution
  
  z <- runif(1,q1,q2)
  z <- qnorm(z,mu,sig)
  if (z==-Inf) z=lo
  if (z==Inf)  z=hi
  z
}
acceptMH <- function(log.pold,log.pnew,param.old,param.new,log.prop.ratio){   #accept for M, M-H
  
  a    <- exp(log.pnew - log.pold + log.prop.ratio)       #acceptance PR
  z    <- runif(1,0,1)
  if (z < a) param.old = param.new #accept
  param.old
}

If you run the code below, you should be able to see step-by-step what the MH algorithm is doing.

rm(list=ls(all=T))
set.seed(1)

setwd('U:/UF/courses/bayesian course/2016/chapter12')
source('useful functions.R')

#our data
y=c(3,10,3,2,4,5);
n=c(10,10,5,4,5,6);

#hyper-prior parameters for theta
a=b=1

#this is for the proposal distribution
sig2.jump=0.06

#initial value
theta=0.6

nsim=10000
store.theta=matrix(NA,nsim,1)

#draw the target distribution T() up to a proportionality constant
z=seq(from=0.01,to=0.99,length.out=10000)
l.prob=rep(NA,length(z))
for (i in 1:length(z)) l.prob[i]=sum(dbinom(y,size=n,prob=z[i],log=T))+dbeta(z[i],a,b,log=T)
par(ask=F)
prob=exp(l.prob)
plot(z,prob,type='l',ylim=c(0,max(prob)),ylab='',xlab='theta')

#start MH algorithm
accept=0
for (i in 1:nsim){
  #draw the FCD up to a proportionality constant
  par(ask=T) 
  plot(z,prob,type='l',ylim=c(0,max(prob)),ylab='',xlab='theta')
  theta.old=theta
  #calculate log of target distribution at theta.old
  log.pold=sum(dbinom(y,size=n,prob=theta.old,log=T))+dbeta(theta.old,a,b,log=T)
  
  #draw T(theta.old)
  points(theta.old,exp(log.pold),pch=19)
  lines(rep(theta.old,2),c(0,exp(log.pold)),lty=3)
  
  #draw proposal distribution
  kk=dnorm(z,mean=theta.old,sd=sig2.jump)
  lines(z,0.1*max(prob)*kk/max(kk),col='grey')
  
  #propose new value
  theta.new=tnorm(lo=0,hi=1,mu=theta.old,sig=sig2.jump)  
  #calculate log of target distribution at theta.new
  log.pnew=sum(dbinom(y,size=n,prob=theta.new,log=T))+dbeta(theta.new,a,b,log=T)
  
  #draw T(theta.new)
  points(theta.new,exp(log.pnew),cex=1.5)  

  #calculate the ratio of the proposal distributions
  num=pnorm((1-theta.old)/sig2.jump)-pnorm((0-theta.old)/sig2.jump) #numerator
  den=pnorm((1-theta.new)/sig2.jump)-pnorm((0-theta.new)/sig2.jump) #denominator
  log.prop.ratio=log(num)-log(den)

  #accept or reject?
  theta=acceptMH(log.pold=log.pold,log.pnew=log.pnew,
                 param.old=theta.old,param.new=theta.new,
                 log.prop.ratio=log.prop.ratio)
  
  #draw blue arrows
  if (theta==theta.new) {
    arrows(theta.old,exp(log.pold),theta.new,exp(log.pnew),col='blue',lwd=2)    
    points(theta.new,exp(log.pnew),cex=1.5,pch=19,col='blue'); 
    accept=accept+1
    lines(rep(theta.new,2),c(0,exp(log.pnew)),lty=3,col='blue')
  }
  #draw red arrows
  if (theta!=theta.new) {
    arrows(theta.old,exp(log.pold),theta.new,exp(log.pnew),col='red',lwd=2)    
    points(theta.new,exp(log.pnew),cex=1.5,pch=19,col='red')  
    lines(rep(theta.new,2),c(0,exp(log.pnew)),lty=3,col='red')    
  }
  store.theta[i]=theta
}

Notice that the acceptance rate is equal to:

accept/nsim

Note that I have made a couple of simplifications in this code to avoid making it too convoluted, including drawing a normal distribution for the proposal distribution when actually it should have been a truncated normal distribution. I urge you to play with this code to see how changing \(\sigma^2_{jump}\) influences the algorithm and the acceptance rate.
